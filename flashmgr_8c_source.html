18 /*
19  * This flash manager is rudimentary. It does wear leveling by allocating small chunks at a time.
20  * When a new parameter block is to be written the old one is erased by writing zeros to it. When
21  * there is no more space in the 1K chunk for a new parameter block the entire chunk is erased.
22  *
23  * assumptions:
24  * -- erased FLASH is all ones
25  * -- bits in individual words can be written to zero
26  * -- a word that is part ones and part zeros can be written to all zeros
27  * -- to get any bits changed back to ones the entire chunk has to be erased
28  *
29  * A parameter block starts with a one word size field. The size indicates the size of the following data.
30  * If the size field is zero, the block is erased.
31  *
32  * A chunk of flash can be in one of three states:
33  * -- erased (all ones)
34  * -- data (starts with a size word that is not 0 or 0xffffffff
35  * -- zeroed (all zeros)
36  *
37  */
38
